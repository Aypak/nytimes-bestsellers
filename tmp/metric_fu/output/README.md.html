3# New York Times Bestsellers
 
4
 
5This is a Ruby wrapper for the [NYTimes Bestsellers API](http://developer.nytimes.com/docs/best_sellers_api).
 
6
 
7## Install
 
8
 
 9Add this line to your application's Gemfile:
10
 
10
 
11    gem 'nytimes-bestsellers'
 
12
 
13And then execute:
 
14
 
15    $ bundle
 
16
 
17Or install it yourself as:
 
18
 
19    $ gem install nytimes-bestsellers
 
20
 
21## Getting Started
 
22
 
23You'll want to sign up for an [API key](http://developer.nytimes.com/apps/register) to start.
24
 
24
 
25Setup a new instance:
 
26
 
27```ruby
28
 
28@client = Bestsellers.new
 
29@client.configure do |config|
 
30  config.api_key = "YOUR_API_KEY"
 
31end
 
32```

34## Methods
 
35
 
36Five requests are available: get a best-seller list, search best-seller lists, get the history of a best seller, get an overview of all of the best-seller lists for a given week and get the names of Times best-seller lists.
 
37
 
381. get_list(list_name, o = {})
 
39
 
40This method has a required parameter of list name(see lists below or call the find_list_names method to find them for yourself) and takes an optional parameter of date, as a string in YYYY-MM-DD format. It also takes an optional parameter of response format and if none is specified it defaults to json.
 
41
 
42with no optional parameters it gets the most recent list
 
43`@client.get_list('hardcover-nonfiction')`
 
44
 
45it can take optional parameters
 
46`@client.get_list('hardcover-nonfiction', date: '2012-04-12', response: 'xml')`
 
47
 
482. bestseller_lists_overview()
 
49
 
50Returns an overview of Times best-seller lists for a single week. Takes an optional parameter of date in YYYY-MM-DD format. If you do not specify a published_date the most recent week's best-seller list will be returned.
51
 
51
 
52`@client.bestseller_lists_overview`
 
53`@client.bestseller_lists_overview(date: '2012-04-12')`
 
54
 
55
 
563. search_list(list_name, o = {})
 
57
 
58Allows for a filtered search of a specific list. Optional parameters are date, isbn, published_date, rank, rank_last_week, and weeks_on_list.
 
59
 
604. single_history
 
61
 
625. find_list_names
 
63
 
64This method takes no parameters and returns all available list names which you can use to query the get_list method.
 
65
 
666. age_groups
 
67
 
68This method takes no parameters and returns all available age groups which you can use to query the single_history method. In most cases the age group will be null, so it's not a great search method on the data. I've listen the age groups below but you can see these available age groups for yourself using the `age_groups` method.
 
69

71## Lists
 
72
 
73The list names to search from are:
 
74
 
75Combined Print and E-Book Fiction, Combined Print and E-Book Nonfiction, Hardcover Fiction, Hardcover Nonfiction, Trade Fiction Paperback, Mass Market Paperback, Paperback Nonfiction, E-Book Fiction, E-Book Nonfiction, Hardcover Advice, Paperback Advice, Advice How-To and Miscellaneous, Picture Books, Chapter Books, Childrens Middle Grade, Young Adult, Paperback Books, Series Books, Hardcover Graphic Books, Paperback Graphic Books, Manga, Combined Print Fiction, Combined Print Nonfiction, Hardcover Business Books, Paperback Business Books, Business Books, Hardcover Political Books, Science Times, Dining
 
76
 
77When passing these names as paraemeters replace spaces with hyphens. The parameter is not case sensitive.
